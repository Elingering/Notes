# 34 | Nginx：高性能的Web服务器
## 进程池
Nginx 则完全不同，“一反惯例”地没有使用多线程，而是使用了“==进程池 + 单线程==”的工作模式。

Nginx 在启动的时候会预先创建好固定数量的 worker 进程，在之后的运行过程中不会再 fork 出新进程，这就是进程池，而且可以自动把进程“绑定”到独立的 CPU 上，这样就完全消除了进程创建和切换的成本，能够充分利用多核 CPU 的计算能力。
